# VISION — AutEng HQ

## What It Is

AutEng HQ is a local-first desktop application that lets high-agency individuals launch, orchestrate, and manage businesses built and operated by AI agents. You provide a starting prompt — HQ turns it into a running business.

## The Problem

Building and running software businesses today requires assembling teams, managing infrastructure, and coordinating across dozens of tools. AI agents can now do most of this work, but there's no command center to orchestrate them — no way to go from "I want to build X" to a live, monitored, KPI-tracked product without stitching together tools manually.

## The Solution

HQ is the cockpit. From a single desktop app, you:

1. **Prompt** — Describe what you want to build
2. **Plan** — HQ generates the AutEng workflow docs (VISION, ARCH, PLAN, TAXONOMY, CODING-STANDARDS) for the new project
3. **Build** — HQ spawns dev agents (Claude Code, Codex CLI) to implement each phase
4. **Deploy** — HQ orchestrates local runs or cloud deployment
5. **Monitor** — Track KPIs, logs, errors, and business metrics from the dashboard
6. **Iterate** — Agents handle maintenance, new features, and scaling

## Target Users

High-agency individuals who want to run one or many agent-operated businesses. Not developers necessarily — operators, founders, and builders who think in outcomes, not code.

### User Profiles

- **Solo operator**: Runs 3-5 micro-SaaS products, each built and maintained by agents
- **Technical founder**: Uses HQ to rapidly prototype and validate ideas before committing personal time
- **Agency owner**: Manages client projects where agents do the building, they do the selling

## Core Principles

1. **Local-first** — HQ runs on your machine. Your data, your control.
2. **Prompt-to-product** — One prompt starts the entire lifecycle.
3. **Agent-native** — Agents are first-class workers, not assistants. They own phases.
4. **Open protocols** — MCP for tool integration, x402 standard APIs everywhere.
5. **Observable** — Every agent action, deployment, and metric is visible in the dashboard.

## Pricing

**Free and open source.** HQ itself costs nothing. Users pay for the services agents consume:


- **AI**: API costs for Claude, Codex, or other models
- **Infrastructure**: Cloud hosting costs for deployed projects (Vercel, AWS, etc.)
- **Tooling**: X402, MCP

HQ is the free cockpit; the engines have their own meters.

## Success Metrics

| Metric | Target |
|--------|--------|
| Time from prompt to deployed v1 | < 1 hour for simple SaaS |
| Projects manageable per user | 10+ concurrently |
| Agent task success rate | > 90% without human intervention |
| User retention (weekly active) | > 60% at 30 days |
| Community contributors | 50+ in first 6 months |

## What HQ Is NOT

- Not an IDE (agents use their own: Claude Code, Codex CLI)
- Not a hosting platform (it deploys to existing platforms)
- Not a no-code builder (it generates real code via agents)
- Not a chat interface (it's a dashboard and orchestrator)
